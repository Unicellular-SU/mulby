# 任务调度器改进说明

## 改进概述

本次改进针对任务调度系统的四个关键问题进行了优化，提升了系统的稳定性、性能和资源利用率。

## 改进内容

### 1. ✅ 插件进程生命周期管理

**问题：** 不支持后台运行的插件，执行完任务后 Host 进程不会被销毁，导致内存占用持续增长。

**解决方案：**
- 采用**智能延迟销毁**策略，符合"临时唤醒"的语义
- 任务执行完成后，检查该插件的下一个任务：
  - 如果下一个任务在 30 秒内执行 → 保持进程（避免频繁创建/销毁）
  - 如果下一个任务超过 30 秒 → 立即销毁
  - 如果没有下一个任务 → 立即销毁
- 既避免了频繁创建/销毁的开销，又及时释放了资源

**代码位置：** `src/main/scheduler/task-scheduler.ts`
- 新增字段：`pluginCleanupTimers`, `CLEANUP_DELAY_SHORT`
- 新增方法：`schedulePluginCleanup()`, `cleanupPlugin()`
- 修改方法：`callPluginCallback()`, `stop()`

**效果：**
```
场景 1：任务密集（间隔 5 秒）
改进前：每次都创建/销毁进程，开销大
改进后：重用进程，最后一个任务执行完 30 秒后销毁

场景 2：任务稀疏（间隔 1 小时）
改进前：进程持续运行，浪费资源
改进后：任务执行完立即销毁，及时释放资源
```

---

### 2. ✅ 智能预启动机制

**问题：** 应用启动时，只有支持后台运行的插件会被启动，不支持后台的插件即使有即将到期的任务也不会预启动，导致任务执行延迟。

**解决方案：**
- 在 `restore()` 方法中添加预启动逻辑
- 检测 1 分钟内要执行的任务
- 后台插件：启动后台进程
- 非后台插件：只初始化 Host（不启动后台）
- 减少任务执行时的初始化延迟

**代码位置：** `src/main/scheduler/task-scheduler.ts`
- 修改方法：`restore()`

**效果：**
```
场景：应用启动时，有个任务在 30 秒后执行
改进前：任务到期时才初始化插件（耗时 2 秒），实际延迟 2 秒执行
改进后：应用启动时预初始化插件，任务准时执行
```

---

### 3. ✅ 并发任务执行

**问题：** 任务是严格串行执行的，如果一个任务执行时间很长，会阻塞后续所有任务。

**解决方案：**
- 添加 `runningTasks` 集合，跟踪正在执行的任务
- 添加 `MAX_CONCURRENT_TASKS` 配置（默认 5 个）
- 修改 `scheduleNext()` 支持并发执行多个到期任务
- 新增 `executeReadyTasks()` 方法，批量启动到期任务
- 任务执行完成后触发下一轮调度

**代码位置：** `src/main/scheduler/task-scheduler.ts`
- 新增字段：`runningTasks`, `MAX_CONCURRENT_TASKS`
- 新增方法：`executeReadyTasks()`
- 修改方法：`scheduleNext()`, `executeTask()`

**效果：**
```
场景：3 个任务同时到期，每个耗时 10 秒
改进前：串行执行，总耗时 30 秒
改进后：并发执行，总耗时 10 秒
```

---

### 4. ✅ 任务优先级支持

**问题：** 所有任务按时间顺序执行，无法区分重要任务和普通任务。

**解决方案：**
- 在 `TaskInput` 和 `Task` 接口中添加 `priority` 字段（0-10，默认 5）
- 修改 `TaskQueue` 的比较逻辑，优先执行高优先级任务
- 更新数据库表结构，添加 `priority` 字段和索引
- 更新 `saveTask()` 和 `rowToTask()` 方法支持优先级

**代码位置：**
- `src/main/scheduler/types.ts` - 类型定义
- `src/main/scheduler/task-queue.ts` - 队列比较逻辑
- `src/main/scheduler/task-store.ts` - 数据库存储

**效果：**
```
场景：两个任务同时到期，一个优先级 8，一个优先级 5
改进前：按时间顺序执行
改进后：优先级 8 的任务先执行
```

---

## 数据库变更

### 新增字段
- `tasks.priority` - 任务优先级（INTEGER，默认 5）

### 新增索引
- `idx_priority` - 优先级索引，提升查询性能

**迁移说明：** 数据库会自动添加新字段，现有任务的优先级默认为 5。

## 配置参数

### 可调整的参数

```typescript
// 插件清理延迟（毫秒）
// 如果下一个任务在此时间内执行，保持进程；否则立即销毁
private readonly CLEANUP_DELAY_SHORT = 30 * 1000  // 30秒

// 最大并发任务数
private readonly MAX_CONCURRENT_TASKS = 5

// 预启动阈值（毫秒）
const PRESTART_THRESHOLD = 60000  // 1分钟
```

### 建议配置

- **低配置设备：** `MAX_CONCURRENT_TASKS = 3`, `CLEANUP_DELAY_SHORT = 15 * 1000`
- **高配置设备：** `MAX_CONCURRENT_TASKS = 10`, `CLEANUP_DELAY_SHORT = 60 * 1000`
- **服务器环境：** `MAX_CONCURRENT_TASKS = 20`, `CLEANUP_DELAY_SHORT = 120 * 1000`

## 已知限制

1. **并发限制：** 最多同时执行 5 个任务（可配置）
2. **清理延迟：** 插件进程最少保持到下一个任务前 30 秒（可配置）
3. **优先级范围：** 0-10，超出范围会被忽略
