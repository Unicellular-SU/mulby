# 附件管理器（方案 A）实现方案

目标：替换现有“横向附件列表”为“摘要条 + 管理器”，在不改变搜索框主布局与插件结果 grid 的前提下，提供批量管理能力，并与附着模式插件窗口协调。

## 版式与结构

1) 搜索框区域（保持单行输入）
- 搜索框高度保持 62px。
- 附件摘要条放在输入框右侧，同一行内展示。
- 摘要条以“附件数量 + 总大小 + 管理/收起”为主，避免挤压输入区。

2) 附件管理器（占用结果列表区域）
- 管理器打开时，替换搜索结果 grid 的区域（同宽、同 x、同 border-top）。
- 高度根据附件数量自适应（最多显示 6 行），内部滚动，不改变搜索框高度。
- 关闭管理器后恢复结果 grid。

3) 附着模式插件窗口适配
- 管理器打开时主动关闭/隐藏附着插件窗口（避免遮挡重叠）。
- 管理器关闭后允许重新附着（与现有“输入变化会关闭面板”的交互一致）。

## 交互与行为

- 摘要条：显示 `附件数量 + 总大小`，chip 支持单项删除。
- 管理器：
  - Header：标题 + 统计 + “清空/关闭”
  - 工具条：全选、删除选中、搜索过滤
  - 列表项：缩略图/图标、名称、大小，右侧勾选 + 单项删除
- 删除动作支持撤销（后续可扩展 Toast + Undo）。
- Esc 优先关闭附件管理器。

## 适配与一致性

- 颜色、圆角、阴影、间距沿用现有变量（`--bg-*`, `--text-*`, `--border`）。
- border-top 与 `.plugin-grid` 对齐，保证视觉一致。
- 触控与可点击目标 >= 32px，按钮与图标不使用 emoji。

## 实现范围（代码）

1) 组件
- 新增 `AttachmentManager` 组件（列表 + 批量操作）。
- `SearchInput` 替换附件横向列表为摘要条。
- `App` 维护附件管理器开关状态，控制窗口高度与结果列表渲染。

2) 状态
- `attachmentsManagerOpen`：管理器开关。
- 打开管理器时关闭附着插件窗口（复用现有关闭逻辑）。
- attachments 为空时自动关闭管理器。

3) 窗口高度计算
- 搜索框高度 = 62 + 摘要条高度（有附件时）。
- 管理器打开时：窗口高度 = 搜索框高度 + 管理器高度 + 1px border。
- 其他场景保持现有逻辑。
