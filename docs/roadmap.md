# Mulby 项目路线图

> 最后更新：2026-02-28
> 当前版本：v0.1.x-dev

## 路线说明
本路线图已根据当前项目状态与产品决策更新，以下事项已明确：

- ✅ 优先投入工程质量、可维护性与核心体验
- ❌ Python 插件运行时：废弃，不再进入近期计划
- ⏸ i18n 多语言：待定，暂不进入近期迭代
- ⏳ 插件商店安全增强（签名/校验链）：后期专题推进

---

## 阶段 A：工程稳定与治理（当前阶段）

### 目标
建立稳定可发布的工程基线，减少回归与维护成本。

### 重点事项
- [ ] 质量门禁打通（`typecheck` / `lint` / `test:unit` / build smoke）
- [ ] 文档治理（盘点、清理、索引、状态同步）
- [ ] CI 建设（最小自动化检查流水线）
- [ ] 超大文件拆分（主进程、preload、设置页、插件管理页）
- [ ] 任务调度器 UI 事件驱动化（替代高频轮询）

---

## 阶段 B：产品体验增强（近期）

### 目标
提升用户日常使用体验，降低配置与运维成本。

### 重点事项
- [ ] 设置中心新增“开机自启动”管理
- [ ] 设置中心新增“更新中心”能力（检查版本、更新入口）
- [ ] 任务调度器体验优化（性能、反馈、可观测性）
- [ ] 插件管理与设置体验持续优化

---

## 阶段 C：生态与商业化（中后期）

### 目标
在稳定基线之上推进生态闭环与商业化能力。

### 候选事项
- [ ] 在线插件商店
- [ ] 用户账号体系
- [ ] 付费/打赏机制
- [ ] 插件签名验证与来源信任体系
- [ ] 企业版能力

---

## 不在近期范围
- Python 插件运行时
- i18n 多语言体系
- 插件商店安全体系（后移到中后期专题）
